# Chavruta review: madrij-app-mvp

## What the owner is getting right

- **Stack choice is conservative**. Vite + React 19 + Tailwind + Supabase is a
  boring, well-trodden path for a 3-day shipping window. dnd-kit is the right
  drag library here — react-beautiful-dnd is unmaintained and Framer Motion's
  Reorder primitive doesn't handle the touch-priority case as cleanly.
- **Mobile-first is non-negotiable** for this audience and the plan reflects
  that. Good.
- **Supabase RLS** is in the plan from the start. RLS-after-the-fact is the
  single biggest source of weekend re-deploys in apps this size.

## What might be missing

- **Focus rings in dark mode.** The Tailwind default focus ring against your
  dark-mode background gets ~3.5:1 contrast, which fails WCAG AA. Override
  with `focus:ring-amber-400` or similar. axe-core won't catch this if you
  only test in light mode.
- **Drag handle aria-label.** The dnd-kit example code omits an aria-label on
  the drag handle. Screen readers announce just "button" with no context.
  Add `aria-label="Reorder reading"` to each handle.
- **Offline / poor connection.** Shul wifi is unreliable. Consider a
  service-worker cache for the reading list so a madrij who's already loaded
  the group can drag-reorder without round-tripping. Not blocking — file as a
  follow-up.

## Decision

**signed_off** — execute the plan as written. Address the two a11y items
before flipping to production. The offline support is a nice-to-have, ship
without it.
